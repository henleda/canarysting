## Why this composite (and not any single arc)
- **Arc 1 has the smallest true gap.** Its centerpiece (real-$ meter climbing vs. flat defender bar) is genuinely built today: `AttackerCost.tsx` renders `RealMeter` bound to `real_attack_cost`/`cap_fraction` with explicit "distinct from the defender-side proxy estimate" copy (verified, AttackerCost.tsx ~lines 60-75). Its only net-new dependency is record/replay — a thin add over the proven `anthropic.FakeClient` + `MessageFromJSON` seam (fake.go:62-75, literally built "to queue deterministic responses with realistic Usage").
- **Arc 2 owns the precision frame** that answers the deepest deployment fear (auto-block nukes real traffic). But its wow — a surviving bystander on the *same host* — is **not on the dashboard today**: `ok_flows` only contains canary-touchers (`maxTier >= 1`, views.go:578). So that proof must come from a **terminal curl from the legit IP returning 200**, not an implied dashboard panel.
- **Arc 3's five-axes-at-once is the most differentiated claim but the densest**, and it rests on two awkward dependencies: AX4/AX5 (`exploitBait`, `opExposure`) are not even *constructed* below `FloorAggressive` (attrition.go:154), and there is no per-axis "firing now" callout. Keep its honesty discipline; demote its headline.

## The single non-negotiable across all three
The RealMeter climbs in **~5-second steps**, not smoothly: the backend polls the tap at `defaultPollInterval = 5 * time.Second` (backend.go:27) while the attacker posts every 500ms (`meter-min-interval`, main.go). Narrate it as **"live, sampled every few seconds."** A technical observer on the screen-share *will* notice a smoothly-climbing claim and ding your credibility at the most important moment.

## Honesty contract (say these out loud — they are trust signals to a burned buyer)
1. **Real vs. proxy, never merged.** `real_attack_cost` = the attacker's own `resp.usage` (real $). `tokens_burned` = a defender-side estimate (chars/4 ×3 for bait), demoted below the time/engagement numbers. Two numbers, side by side, never summed. The UI already enforces this; the presenter must too.
2. **Replay is a replay.** The recorded run's `Usage` is from a real prior Opus 4.8 run, so the **dollars are real but the run is a recording.** Say "faithful replay of a real run" before showing it, then prove it live. NEVER show `-scripted` with a real-$ meter — `-scripted` is a zero-API HTTP walk that carries **no Usage** (main.go), so a $ figure on it would be fabrication.
3. **--aggressive is a demo posture.** It trips Tier 2 on the *first* touch for speed (set server-side via `set-demo-posture.sh aggressive`). The realistic default is 3-5 touches. Say so at the escalation beat. Revert posture after (`set-demo-posture.sh default`) so the live M7 learning window isn't contaminated.
4. **Zero false positives is STRUCTURAL, not a rate.** Only a canary touch arms a response — there is no anomaly/deviation classifier to tune, so there is no FP rate to quote or to drift (Rule 8). Pre-empt this in the first 45 seconds. Pin it to "zero by construction; here is the code path," never a percentage.
5. **Pre-empt the killer unasked question** — "what about an attacker who never touches a canary?" Honest answer: then we never act (that *is* the no-bystander-harm guarantee); the bet is comprehensive decoy coverage in the negative space + recon early-warning surfacing the probing before a touch. Do **not** overclaim coverage.
6. **Cross-customer is now a LIVE demo (06-14 update).** The k=3 crossing is real: 3 boxes each independently jail the same coarse pattern → the aggregator rejects at k=2, crosses at k=3; `cat` the wire = an opaque token + 7 coarse fields (no raw traffic/baselines/decoy contents). DISCLOSE: these are 3 staged scopes WE operate (not 3 customers); the identical pattern is by construction (one replayed cassette); k≥3 = operator-vouched threshold-ENFORCEMENT, not Sybil-resistance (that's D7); the crossing reaches the internal consume surface, not the external feed (FeedK=5 unmet). Never say "three customers independently saw this."
9. **A capable AI attacker WILL detect a safe deception — and that's fine (the harmlessness floor).** The `EXAMPLE` keys, reserved TLDs, and tarpit/429 signature are the safety guarantee (provably-fake = detectable); we never ship real-looking secrets. Do NOT claim we fool a sophisticated attacker. The honest claim: we DETECT it (a zero-FP alert), DELAY + DENY it (real cost + time, zero real data), and turn it into INTEL. Detection-then-disengage is the deception SUCCEEDING — the attacker left empty-handed having tripped a silent alarm and donated its TTPs.
7. **The disengage signal must be real.** `DisengagedEarly` is incremented only on `contract.DisengageAttacker` (cost.go:114-115). If the recorded run was defender-capped, narrate it as capped — never relabel a max-hold cap as the attacker giving up.
8. **The attacker is our own declared-IP harness**, a faithful adversary emulation against a known range — not a captured wild attacker. Still a strong, honest proof; say which it is.
